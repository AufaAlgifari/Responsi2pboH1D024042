Pertanyaan Refleksi
Setelah menyelesaikan kode, jawab pertanyaan berikut:

Apa yang dimaksud dengan constructor overloading dalam pemrograman berorientasi objek?
Bagaimana cara Java membedakan constructor yang memiliki nama sama namun parameter berbeda?
Apa perbedaan antara mengakses variabel secara langsung (misalnya: objek.variabel = nilai) dengan menggunakan method setter?
Jelaskan konsep encapsulation dan bagaimana method dapat membantu melindungi data dalam sebuah class!
Mengapa constructor tidak memiliki return type, bahkan tidak menggunakan kata kunci void?

Jawaban

constructor overloading adalah konsep di mana sebuah kelas memiliki lebih dari satu
constructor dengan nama yang sama, tetapi parameter berbeda.

Java akan memilih constructor yang sesuai dengan argumen yang diberikan saat objek dibuat.
Ini disebut polymorphism pada level constructor.

Mengakses variabel secara langsung, berarti kita mengubah data secara langsung dari luar kelas,
Sementara menggunakan method setter memberi kesempatan untuk menambahkan aturan atau
pengecekan sebelum data diubah, sehingga lebih aman dan terkontrol.

konsep encapsulation adalah prinsip OOP di mana data (variabel) dalam sebuah kelas dijaga kerahasiaannya dengan
membuatnya private, dan hanya bisa diakses atau diubah melalui method publik seperti getter dan setter.
Metode ini membantu melindungi data dari perubahan yang tidak diinginkan dan memastikan data selalu valid.

Karena tujuan constructor bukan untuk mengembalikan nilai, melainkan menginisialisasi objek. Java otomatis mengembalikan
referensi ke objek yang dibuat, jadi tidak perlu menuliskan return type.
